2. Planung bis zum nächsten Meeting am Sonntag

	2.1 Besprechung der Startschwierigkeiten des GUI
		- Da die GUI-Anwenung im Moment noch nicht korrekt startet, müssen erst die Startschwierigkeiten überwunden werden, bevor die eigentliche Implementierung anfangen kann.
		  Hier wird noch auf Antwort von Georg gewartet.
		  
	2.2 Umsetzung des Spielfelds
		- Das Spielfeld wird dynamisch auf Usereingabe aufgebaut. Eine genaue Planung ist aufgrund mangelnder Erfahrungen schwer möglich, doch die Idee ist wie folgt:
		Der Spieler soll eine Größe wählen (z.B. via Dropdownmenu) und anschließend werden in einer Forschleife entsprechend viele Tiles/Buttons einer Liste hinzugefügt. 
		Hier werden je zwei das gleiche Bild zugeordnet. Die Anordnung im GUI erfolgt dann nachdem die Tiles-Liste geshuffeld wurde, wie bisher in der Konsolenanwendung.
		
	2.3 Besprechung welche Design Patterns noch umgesetzt werden
		- Bis jetzt wurden HighScoreHistory, Database und Controller als Singleton umgesetzt. Bei Controller kam es dadurch zu Problemen mit dem GUI - hier muss noch nachgearbeitet werden.
		- Evtl. sollen die Karten über das Factory Pattern erstellt werden.
		- Evlt. könnte das Spielfeld mit dem Observer Pattern umgesetzt werden.
